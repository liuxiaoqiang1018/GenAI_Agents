## 学习方式

1. 先跑通 demo，再标注核心概念位置，理解调用关系
2. 喜欢看 diff 形式的代码修改，便于理解每一步变化
3. 用 Java 类比理解 Python 框架（如 LangGraph ≈ Spring State Machine）
4. 每课必须有"透明版"代码（main_debug.py），打印 Agent 每个阶段的完整 Prompt 和调用日志

## 核心领悟

> **"提示词(prompt)在主导 Agent，提示词在发力"**

学完 28 课后总结出的本质认知：
- Agent = system prompt
- Task = user prompt
- 路由 / 反思 / 协作 = 不同的 prompt 组织方式
- 所有框架本质都是管理 prompt 的拼接顺序和调用流程
- **框架只是脚手架，prompt 才是核心驱动力**
