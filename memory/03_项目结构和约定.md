# 项目结构和约定

## 项目概述

GenAI_Agents 是一个生成式 AI 代理教程学习项目，基于 Nir Diamant 的 46+ Jupyter Notebook 教程，我逐课学习并提取为可运行的 Python 脚本。

## 目录结构

```
GenAI_Agents/
├── all_agents_tutorials/    # 原始 Jupyter Notebook（核心内容）
│   ├── scripts/             # 辅助脚本（如 mcp_server.py）
│   └── data/                # 笔记本专用数据
├── md/                      # 我的学习产出（按课编号）
│   ├── 01/
│   │   ├── 第1课_xxx.md     # 中文教程文档
│   │   ├── main.py          # 使用框架的正式代码
│   │   ├── main_debug.py    # 不用框架的调试演示代码
│   │   ├── .env             # 环境变量配置（所有课程的源头）
│   │   └── venv/            # 独立虚拟环境
│   ├── 02/
│   ├── ...
│   └── 28/
├── java/md/                 # Java 框架对照文档
├── memory/                  # 本参考文档（你正在看的）
├── data/                    # 项目级数据文件
├── images/                  # 架构图（SVG）
└── CLAUDE.md                # Claude Code 项目指令
```

## 每课学习流程

1. 翻译原始教程为中文 md，放到 `md/编号/` 目录下
2. 提取代码到 `main.py`，配置 `.env`，创建独立 venv 跑通
3. 标注 Agent / Tool / LLM 在代码中的位置
4. 用 Java 类比解释调用关系
5. 动手改造 demo（如加入自定义 Tool），以 diff 形式展示每步修改
6. 创建 `main_debug.py`：不用任何框架，纯手写 Agent 流程，打印每个阶段的完整 Prompt 和调用日志

## 文件说明

| 文件 | 说明 |
|---|---|
| `main.py` | 使用框架的正式代码，也要打印关键交互过程 |
| `main_debug.py` | 纯手写版，打印 Agent 每个阶段的完整 Prompt 和调用日志，用于理解框架内部机制 |
| `第X课_xxx.md` | 中文教程文档，包含依赖安装说明 |
| `.env` | 环境变量，从 `md/01/.env` 同步 |

## 技术栈

- **语言**：Python（Jupyter Notebooks + 提取脚本）
- **主要框架**：LangChain, LangGraph, PydanticAI
- **其他框架**：CrewAI, AutoGen, OpenAI Swarm, MCP
- **LLM**：通过 OpenAI 兼容接口调用（公益站 / MiniMax / Kimi 等）
