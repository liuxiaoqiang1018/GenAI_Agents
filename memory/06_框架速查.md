# AI Agent 框架速查

从 28 课学习中提炼的核心概念，每个概念一句话解释 + Java 类比。

---

## 核心概念

| 概念 | 一句话解释 | Java 类比 | 相关课程 |
|---|---|---|---|
| **Agent** | 一个有 system prompt 的 LLM，能根据指令自主决策下一步动作 | Controller/Service | 全部课程 |
| **Tool / Function Calling** | 给 LLM 注册可调用的函数，LLM 决定何时调用、传什么参数 | 接口方法 + 反射调用 | 第3、5、25课 |
| **Chain** | 多个步骤顺序串联执行（A→B→C） | Stream 链式操作 | 第2课 |
| **Graph / 图** | 多个节点 + 边组成的工作流，支持循环和分支 | Spring State Machine / Flowable | 第4课起 |
| **State / 状态** | 在图的节点之间传递的共享数据容器 | ThreadLocal / Context | 第4课起 |
| **Router / 路由** | 根据条件决定走哪条分支的节点 | if-else / 策略模式 | 第9、12课 |
| **Memory / 记忆** | 让 Agent 记住历史对话，分短期（上下文窗口）和长期（向量库） | Session / Redis 缓存 | 第1、22课 |
| **RAG** | 检索增强生成：先从知识库检索相关内容，再交给 LLM 回答 | 先查数据库再拼模板 | 第2、7课 |
| **ReAct** | 思考→行动→观察→再思考的循环模式，Agent 的核心运行方式 | while 循环 + 策略模式 | 第7、25课 |
| **Human-in-the-Loop** | 在 Agent 工作流中插入人工审批/确认节点 | 工单审批流程 | 第8课 |
| **Multi-Agent** | 多个 Agent 协作完成任务，各有不同角色和专长 | 微服务编排 | 第6、23、27课 |
| **Self-Reflection** | Agent 自己评估输出质量，不满意就重来 | 单元测试 + 重试 | 第7、13、24课 |
| **Map-Reduce** | 并行处理多个子任务，最后汇总结果 | Fork/Join、CompletableFuture | 第14、19课 |
| **子图嵌套** | 一个大图的某个节点本身是另一个完整的图 | 方法调用嵌套 / 子流程 | 第21课 |
| **向量数据库** | 将文本转为向量存储，用于相似性搜索（如 ChromaDB） | Elasticsearch / 全文检索 | 第30课 |
| **自愈循环** | 代码执行→捕获错误→LLM修复→热补丁→重新执行 | try/catch + 热部署 | 第30课 |
| **Supervisor 模式** | 一个主管 Agent 持有多个子 Agent 引用并调度协作 | 服务编排层 / Orchestrator | 第31课 |
| **计划-执行分离** | 先用 LLM 生成执行计划，再逐步执行 | 策略模式 + 命令模式 | 第13、31课 |
| **三层记忆** | 情景(案例Few-Shot)+语义(偏好)+程序(可更新Prompt) | 案例库+用户画像+配置中心 | 第32课 |
| **Prompt自我优化** | 根据反馈用 LLM 改写 Prompt，存回记忆下次使用 | 热更新配置 | 第32课 |

## 框架速查

| 框架 | 定位 | 核心特点 |
|---|---|---|
| **LangChain** | 基础框架 | Chain + Prompt + Tool + Memory 一站式 |
| **LangGraph** | LangChain 的图扩展 | 有状态的循环/分支工作流，生产级 Agent 的首选 |
| **PydanticAI** | 轻量结构化框架 | 类型安全、结构化输出、依赖注入 |
| **CrewAI** | 多 Agent 协作 | Agent/Task/Crew 三层结构，角色扮演式 |
| **AutoGen** | 多 Agent 对话 | 群聊模式、发言权转移、Manager 选人 |
| **MCP** | 工具协议标准 | Server/Client/Host 架构，标准化工具接入 |
| **OpenAI Swarm** | 轻量多Agent框架 | Agent接力(Handoff)，LLM自主决定何时转交 |

---

## 核心领悟

> **所有框架本质都是管理 prompt 的拼接顺序和调用流程。Prompt 主导 Agent，框架只是脚手架。**
