# 第37课：播客自动生成代理（Podcast Generation Agent）

## 课程概述

本课构建一个**全自动播客脚本生成系统**：给定一个主题，自动规划子话题 → 对每个子话题进行"主持人-专家"多轮模拟访谈 → 搜索 Web 和 Wikipedia 获取真实信息 → 将访谈整理为播客脚本 → 最终生成完整播客（开场 + 正文 + 结语）。

**核心模式**：主图 + 子图嵌套 + Send API 并行 + 多轮对话循环 + Map-Reduce 汇总

## 架构流程

```
【主图】
主题 → [规划子图] 生成关键词+子话题
          ↓
     [Send API 并行] 对每个子话题启动一个访谈子图
          ↓ ↓ ↓ (并行)
     [访谈子图 x N] 每个子话题独立的多轮访谈
          ↓
     汇聚所有访谈脚本
          ↓
     [并行] 写开场 / 写正文 / 写结语
          ↓
     合并最终播客脚本 → 结束

【规划子图】
主题 → 生成关键词 → 生成子话题结构 → 返回

【访谈子图】（多轮循环）
主持人提问 → [并行] Web搜索 + Wiki搜索
                   ↓
            专家回答（基于搜索结果）
                   ↓
            [对话够了？]
            ├─ 否 → 主持人继续提问（循环）
            └─ 是 → 保存访谈 → 整理为脚本 → 返回
```

## 关键概念标注

| 概念 | 在代码中的位置 | Java 类比 |
|---|---|---|
| **子图嵌套** | `plan_builder.compile()` 作为主图的一个节点 | 子流程 / 方法调用 |
| **Send API 并行** | `Send("Create podcast", {...})` 对每个子话题并行启动子图 | CompletableFuture.allOf() |
| **多轮访谈循环** | `route_messages` 判断对话是否结束 | while + 终止条件 |
| **并行搜索** | Web搜索和Wiki搜索同时执行 | Fork/Join |
| **Map-Reduce** | 各子话题并行访谈(Map) → 汇总写报告(Reduce) | Stream.parallel().collect() |
| **MessagesState** | 访谈子图用 messages 维护对话历史 | 对话上下文 |
| **结构化输出** | `keywords(BaseModel)` + `with_structured_output` | POJO 映射 |

## 本课特点（对比之前课程）

1. **三层图嵌套**：主图包含规划子图和访谈子图，访谈子图内部还有循环
2. **Send API 并行**：第一次用 Send 对多个子话题并行启动独立的访谈（类似第14课的并行Send）
3. **模拟访谈**：主持人和专家的多轮对话，专家基于实时搜索结果回答
4. **三路并行写作**：开场、正文、结语三个节点并行生成，最后合并
5. **复杂度最高**：到目前为止架构最复杂的一课
