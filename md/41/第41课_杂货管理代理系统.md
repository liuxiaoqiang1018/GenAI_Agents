# 第41课：杂货管理代理系统（Grocery Management Agents System）

## 课程概述

本课使用 **CrewAI** 框架构建一个**家庭杂货管理系统**：从购物小票提取商品信息 → 在线查询保质期 → 根据用户消耗更新库存 → 推荐食谱利用剩余食材。四个 Agent 各有角色和性格，通过 Task 链接力协作。

**核心模式**：CrewAI Agent/Task/Crew 模式 + 上下文传递(context) + 人工输入(human_input) + 网站搜索工具

## 架构流程

```
购物小票(Markdown) → [小票解析Agent] 提取商品/数量/单位
                         ↓ (context传递)
                    [保质期Agent] 在线搜索保质期，估算过期日
                         ↓ (context传递)
                    [库存跟踪Agent] 用户输入已消耗的 → 更新库存
                         ↓ (context传递 + human_input)
                    [食谱推荐Agent] 根据剩余食材搜索食谱
                         ↓ (输出JSON)
                    保存结果 → 结束
```

## 四个 Agent 角色

| Agent | 角色 | 工具 | 特点 |
|---|---|---|---|
| **小票解析者** | 从Markdown小票提取商品信息 | 无 | 精确、细致 |
| **保质期专家** | 在线查询食品保质期 | WebsiteSearchTool(stilltasty.com) | 严谨、可靠 |
| **库存跟踪者** | 根据消耗更新库存 | 无 | human_input=True |
| **食谱推荐者** | 搜索食谱利用剩余食材 | WebsiteSearchTool(americastestkitchen.com) | 创意、实用 |

## 关键概念标注

| 概念 | 在代码中的位置 | Java 类比 |
|---|---|---|
| **Agent(角色)** | `Agent(role, goal, backstory, personality)` | 有职责描述的 Service |
| **Task(任务)** | `Task(agent, description, expected_output, context)` | 有输入输出规范的工作单元 |
| **Crew(团队)** | `Crew(agents, tasks)` → `crew.kickoff()` | 任务编排器 |
| **context 传递** | `context=[read_receipt_task]` — 任务间传递结果 | 方法返回值传参 |
| **human_input** | `human_input=True` — 运行中等待用户输入 | Scanner.nextLine() |
| **WebsiteSearchTool** | 搜索指定网站内容 | Jsoup 定向爬取 |
| **expected_output** | JSON 格式的期望输出模板 | 接口文档/契约 |
| **output_file** | 结果自动保存到文件 | FileWriter |

## 本课特点（对比第28课 CrewAI）

1. **context 链接**：每个 Task 的 context 引用上一个 Task，形成数据管道
2. **human_input**：库存跟踪和食谱推荐两个 Task 支持运行中人工输入
3. **WebsiteSearchTool**：定向搜索特定网站（保质期网站、食谱网站）
4. **expected_output**：用 JSON 模板约束输出格式（类似结构化输出）
5. **personality**：Agent 有"性格"描述，影响回答风格
