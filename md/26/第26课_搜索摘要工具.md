# 第26课 - 搜索摘要工具（搜索→解析→逐条摘要→合并）

## 环境准备

```bash
pip install httpx python-dotenv
```

## 概述

本课构建一个**搜索摘要工具**，自动搜索信息并生成精炼摘要：

1. **搜索** → 根据用户查询获取多条搜索结果
2. **解析** → 解析搜索结果为结构化数据（标题、摘要、链接）
3. **逐条摘要** → 对每条搜索结果独立摘要
4. **合并** → 汇总所有摘要为最终报告

核心模式：**检索增强生成（RAG的简化版）** —— 先搜索外部知识，再用LLM加工。

## 架构

```
用户查询
   │
   ▼
┌──────────┐
│  搜索     │  → 多条结果 [{title, snippet, url}]
└──────────┘
   │
   ├──→ 结果1 → 摘要1
   ├──→ 结果2 → 摘要2     ← 逐条处理（Map）
   ├──→ 结果3 → 摘要3
   │
   ▼
┌──────────┐
│  合并     │  → 最终摘要报告（Reduce）
└──────────┘
```

## 核心概念

### 1. 搜索作为数据源

原始 notebook 用 DuckDuckGo 搜索。我们用 LLM 模拟搜索结果（LLM 本身有大量知识），生产环境中可替换为：
- DuckDuckGo / Bing / Google API
- Tavily（AI 友好的搜索API）

### 2. 逐条摘要 vs 整体摘要

本课对**每条结果**独立摘要，而不是把所有结果拼一起再摘要：
- 逐条摘要：每条 1-2 句，精准保留来源
- 整体摘要：可能丢失来源归属

### 3. 与前几课的关系

| 维度 | 第14课（Map-Reduce） | 第19课（多平台） | **第26课（搜索摘要）** |
|------|-------------------|----------------|---------------------|
| 数据来源 | 内置条款库 | 用户文章 | **搜索引擎** |
| 处理方式 | 并行Send | 并行Send | **逐条摘要** |
| 输出 | 审查报告 | 平台内容 | **摘要报告** |

## 关键洞察

1. **搜索+LLM = 知识增强**：LLM 的知识有时效性，搜索补充最新信息
2. **逐条处理保留来源**：每条摘要带出处，比混在一起更可信
3. **这是 RAG 的简化版**：完整的 RAG 用向量数据库检索，本课用搜索引擎检索，原理一样
