# 第38课：智能购物推荐代理（ShopGenie）

## 课程概述

本课构建一个**智能购物推荐系统**：用户输入查询（如"1000美元以下最好的手机"），系统自动搜索网页 → 提取产品结构化信息 → LLM 对比分析 → 推荐最佳产品 → 搜索 YouTube 评测视频 → 发送推荐邮件。

**核心模式**：搜索-提取-对比-推荐 线性流水线 + 结构化输出(Pydantic) + 多源信息整合

## 架构流程

```
用户查询 → Tavily搜索博客 → 爬取博客全文
    → LLM结构化提取产品信息(Pydantic Schema)
    → LLM对比分析 + 推荐最佳产品
    → YouTube搜索评测视频
    → 展示结果 + 发送推荐邮件 → 结束
```

## 关键概念标注

| 概念 | 在代码中的位置 | Java 类比 |
|---|---|---|
| **结构化提取** | `SmartphoneReview(BaseModel)` — LLM 从博客中提取产品信息 | JSON → POJO 反序列化 |
| **嵌套 Pydantic** | `ProductComparison` 包含 `SpecsComparison` + `RatingsComparison` | 嵌套 DTO |
| **产品对比** | `product_comparison_node` — LLM 对比多个产品并推荐最佳 | 比较器 / 排序器 |
| **多源整合** | Web搜索 + YouTube API + 邮件 | 多数据源聚合 |
| **HTML 邮件模板** | `email_html_template` — 精美格式的推荐邮件 | 邮件模板引擎 |
| **线性流水线** | 6 个节点顺序串联 | Pipeline |

## 本课特点

1. **复杂 Pydantic 结构**：嵌套多层的产品数据模型，LLM 一次性提取完整结构化信息
2. **多源信息整合**：Web 博客 + YouTube 视频 + 邮件推送，三个外部系统协作
3. **HTML 邮件**：生成精美的 HTML 格式推荐邮件（非纯文本）
4. **真实购物场景**：从搜索到推荐到通知的完整购物体验闭环
