# 第20课 - 企业梗图生成器（外部数据采集 + 结构化流水线）

## 环境准备

```bash
pip install langgraph langchain-core httpx python-dotenv
```

`main_debug.py` 只需要 `httpx` 和 `python-dotenv`。

## 概述

本课构建一个**企业梗图生成器**，自动为企业生成品牌相关的梗图文案：

1. **信息采集** → 收集企业/产品的基本信息
2. **企业分析** → 提取品牌调性、目标受众、核心卖点
3. **概念生成** → 生成3个梗图创意概念
4. **模板匹配** → 为每个概念匹配合适的梗图模板
5. **文案生成** → 生成梗图的上下文字（顶部/底部文字）
6. **输出展示** → 展示最终梗图文案

核心模式：**外部数据采集→结构化分析→创意生成**，一条线性流水线但中间数据高度结构化。

## 架构

```
企业信息（用户输入/网页抓取）
   │
   ▼
┌──────────┐
│ 企业分析  │  → {调性, 受众, 卖点, 产品, 个性}
└──────────┘
   │
   ▼
┌──────────┐
│ 概念生成  │  → [{消息, 情绪, 受众关联}] × 3
└──────────┘
   │
   ▼
┌──────────┐
│ 模板匹配  │  → [{模板, 概念, 行数}] × 3
└──────────┘
   │
   ▼
┌──────────┐
│ 文案生成  │  → [{上文, 下文}] × 3
└──────────┘
   │
   ▼
┌──────────┐
│ 输出展示  │
└──────────┘
```

## 核心概念

### 1. 结构化中间数据

每一步的输出都是**结构化的 JSON**，不是自由文本：

```python
# 第2步输出：企业分析（结构化）
{
    "tone": "专业但幽默",
    "target_audience": "程序员",
    "value_proposition": "AI提效",
    "key_products": ["Copilot", "Agent"],
    "brand_personality": "极客范"
}

# 第3步输出：梗图概念（结构化列表）
[
    {"message": "加班到崩溃", "emotion": "自嘲", "audience_relevance": "程序员共鸣"},
    ...
]
```

**Java 类比**：像 DTO 在 Service 层之间传递——不是传 String，而是传强类型对象。

### 2. 外部数据作为输入

原始 notebook 用 `WebBaseLoader` 抓取公司网站。我们简化为用户输入，但模式相同：
- **外部数据** → 进入流水线
- LLM 负责**理解和结构化**外部数据
- 后续步骤基于**结构化数据**而非原始文本

### 3. 与前几课的对比

| 维度 | 第16课（GIF） | 第18课（音乐） | **第20课（梗图）** |
|------|-------------|-------------|-----------------|
| 输入来源 | 用户描述 | 用户描述 | **外部数据/用户输入** |
| 中间数据 | 自由文本 | 自由文本 | **结构化JSON** |
| 创意阶段 | 角色→剧情→提示词 | 旋律→和声→节奏 | **分析→概念→模板→文案** |

## 关键洞察

1. **结构化 > 自由文本**：中间数据用 JSON 传递，后续步骤解析更精确、不会丢信息
2. **分析和创意分离**：先客观分析企业（第2步），再发挥创意（第3-5步），避免"乱编"
3. **梗图模板是约束**：模板限定了文字位置和行数，LLM 必须在约束内创作
4. **线性虽简单但够用**：6步顺序执行，不需要分支或循环，说明不是所有场景都需要复杂图结构
