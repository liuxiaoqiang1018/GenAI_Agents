```bash
pip install httpx python-dotenv
```

## 概述

本课构建一个**工具调用Agent**，展示从函数到工具到Agent的完整链路：

1. **定义函数** → 普通 Python 函数（摘要、翻译）
2. **包装为工具** → 给函数加上名称、描述、参数schema
3. **Agent调用** → LLM 根据任务自主决定调用哪个工具、传什么参数

核心模式：**Tool Calling** —— LLM 不直接执行操作，而是输出"我要调用哪个工具"的指令，由系统执行后把结果返回给LLM。

## 架构

```
用户输入："请摘要这段文字并翻译成日语"
   │
   ▼
┌──────────────────────────────────────┐
│              LLM 推理                 │
│  "我需要先调用摘要工具，再调翻译工具"   │
│                                      │
│  输出: tool_call("summarize", text)  │
└──────────────────────────────────────┘
   │
   ▼ 系统执行工具
┌──────────────┐
│ summarize()  │ → 摘要结果
└──────────────┘
   │
   ▼ 结果返回给LLM
┌──────────────────────────────────────┐
│              LLM 继续推理              │
│  "摘要完成，现在调翻译工具"            │
│                                      │
│  输出: tool_call("translate", summary)│
└──────────────────────────────────────┘
   │
   ▼ 系统执行工具
┌──────────────┐
│ translate()  │ → 翻译结果
└──────────────┘
   │
   ▼ 结果返回给LLM
┌──────────────────────────────────────┐
│  LLM 生成最终回复                     │
└──────────────────────────────────────┘
```

## 核心概念

### 1. 函数→工具（Function → Tool）

普通函数加上元信息，就变成LLM可调用的"工具"：

```python
# 普通函数
def summarize(text): return call_llm(f"摘要: {text}")

# 包装为工具（加名称+描述+参数schema）
tools = {
    "summarize": {
        "func": summarize,
        "description": "对文本进行摘要",
        "params": {"text": "要摘要的文本"}
    }
}
```

**Java 类比**：
```java
// 像 Spring MVC 的 @RequestMapping
// 普通方法 → 加注解 → 变成可被调度的端点
@Tool(name="summarize", description="对文本进行摘要")
public String summarize(@Param("text") String text) {
    return llm.invoke("摘要: " + text);
}
```

### 2. Agent循环（Agent Loop）

LLM不是一次性给出答案，而是**循环决策**：

```
while True:
    action = llm.decide(task, tool_results)
    if action == "final_answer":
        return answer
    result = execute_tool(action.tool, action.args)
    tool_results.append(result)
```

| 维度 | 第7课（科研助手） | **第25课（工具调用）** |
|------|-----------------|---------------------|
| 工具来源 | LangChain内置 | **自定义Python函数** |

| 重点 | Agent循环 | **函数→工具的包装过程** |

## 关键洞察

1. **工具 = 函数 + 元信息**：名称、描述、参数schema让LLM知道"有什么工具可用、怎么用"
2. **LLM不执行，只决策**：LLM输出"调summarize"，系统负责实际执行
3. **Agent循环 = while + LLM决策 + 工具执行**：直到LLM认为任务完成
