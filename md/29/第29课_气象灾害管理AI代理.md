# 第29课：气象灾害管理 AI 代理

## 课程概述

本课构建一个**气象灾害应急管理系统**，通过 LangGraph 编排多步骤工作流：获取天气数据 → 分析灾害类型 → 评估严重程度 → 根据条件路由到不同部门响应 → 人工审批 → 发送邮件告警。

**核心模式**：条件路由 + Human-in-the-Loop + 多部门分发

## 架构流程

```
获取天气 → 社交媒体监控 → 分析灾害类型 → 评估严重程度 → 数据日志记录
                                                                    ↓
                                                              [条件路由]
                                                    ↙          ↓          ↘
                                          紧急响应    市政工程响应    民防响应
                                            ↓          ↓              ↓
                                        发送邮件    人工审批        人工审批
                                            ↓          ↓              ↓
                                          结束    [审批路由]      [审批路由]
                                               ↙       ↘       ↙       ↘
                                          发送邮件  拒绝处理  发送邮件  拒绝处理
                                              ↓        ↓        ↓        ↓
                                            结束      结束      结束      结束
```

**路由规则**：
- 严重程度 Critical/High → 紧急响应（自动发邮件，不需人工审批）
- 灾害含 flood/storm → 市政工程响应（需人工审批）
- 其他 → 民防响应（需人工审批）

## 关键概念标注

| 概念 | 在代码中的位置 | Java 类比 |
|---|---|---|
| **State（状态）** | `WeatherState(TypedDict)` — 8个字段在节点间传递 | POJO / DTO |
| **Node（节点）** | 每个函数如 `get_weather_data()`, `analyze_disaster_type()` | Service 方法 |
| **Conditional Edge（条件边）** | `route_response()` 根据灾害类型和严重程度路由 | 策略模式 / Router |
| **Human-in-the-Loop** | `get_human_verification()` — 低/中级别需人工输入 y/n | 审批工单 |
| **Prompt Template** | `ChatPromptTemplate.from_template()` — 让 LLM 分析灾害 | String.format() |

## 本课特点（对比之前课程）

1. **真实 API 集成**：调用 OpenWeatherMap API 获取实时天气（之前课程多用模拟数据）
2. **多部门路由**：一个条件路由节点分发给 3 个不同部门（之前最多 2 路分支）
3. **二级路由**：人工审批后再路由到"发邮件"或"拒绝处理"（路由嵌套路由）
4. **邮件通知**：SMTP 发送告警邮件（第一次涉及外部通知系统）
5. **定时调度**：`schedule` 库实现定时轮询（类似 Java 的 ScheduledExecutorService）

## 与简单代理的对比

| 维度 | 简单 Agent（如第2课） | 本课气象灾害系统 |
|---|---|---|
| 流程控制 | 线性（问→答） | 多级条件路由 + 人工审批 |
| 数据来源 | 用户输入 | 真实 API + 社交媒体模拟 |
| 输出方式 | 控制台打印 | 邮件告警 + 日志文件 |
| LLM 调用次数 | 1 次 | 3-4 次（分析、评估、生成响应计划） |
| 人工介入 | 无 | 低/中级别需人工 y/n 审批 |
