# 第15课 - E2E测试代理（动作循环 + 代码生成验证）

## 环境准备

```bash
pip install langgraph langchain-core httpx python-dotenv
```

`main_debug.py` 只需要 `httpx` 和 `python-dotenv`。

注：原始 notebook 使用 Playwright 控制浏览器，本课简化为**API测试代码生成**，聚焦核心模式。

## 概述

本课构建一个**E2E测试代理**，用户用自然语言描述测试需求，Agent 自动：

1. **指令拆解** → 把自然语言测试描述拆成原子动作列表
2. **逐个生成代码** → 为每个动作生成 Python 测试代码
3. **语法验证** → 用 `ast.parse()` 检查代码是否合法
4. **条件路由** → 验证通过→下一个动作；失败→错误处理
5. **组装脚本** → 把所有动作代码拼成完整测试函数
6. **执行测试** → 运行生成的测试代码
7. **生成报告** → 汇总测试结果

核心模式：**带计数器的动作循环** —— 逐个处理动作列表，每个动作经过 生成→验证→(成功则继续/失败则处理错误) 的流程。

## 动机

传统 E2E 测试需要手写大量代码。用 AI Agent：
- 用自然语言描述"注册表单填写→提交→验证成功"
- Agent 自动生成对应的测试脚本
- 逐步生成，每步验证，出错及时处理

这个模式适用于所有"**逐步生成+逐步验证**"的场景：代码生成、文档写作、数据处理管道等。

## 核心概念

### 1. 带计数器的动作循环（Action Loop with Counter）

用 `current_action` 计数器跟踪当前处理到第几个动作：

```python
def decide_next(state):
    if state["error_message"]:
        return "handle_error"           # 验证失败
    elif state["current_action"] >= len(state["actions"]):
        return "post_process"           # 全部完成
    else:
        return "generate_code"          # 继续下一个
```

**Java 类比**：
```java
for (int i = 0; i < actions.size(); i++) {
    String code = codeGenerator.generate(actions.get(i));
    if (!validator.isValid(code)) {
        return handleError(code);  // 失败直接退出
    }
    script.append(code);
}
return postProcess(script);
```

### 2. 代码生成+验证（Code Generation + Validation）

LLM生成的代码不一定语法正确，用 `ast.parse()` 做语法检查：

```python
import ast
try:
    ast.parse(generated_code)
    # 语法正确，继续
except SyntaxError as e:
    # 语法错误，进入错误处理
    state["error_message"] = str(e)
```

### 3. 三路条件路由（Three-Way Routing）

验证后有3条路径，这是之前没见过的三路分支：
- **通过+继续** → 回到生成代码（循环）
- **通过+完成** → 进入后处理
- **失败** → 进入错误处理

### 4. 与前几课的对比

| 维度 | 第7课（Agent循环） | 第13课（自反思） | **第15课（动作循环）** |

## 关键洞察

1. **动作循环 = 带索引的 for 循环**：用 `current_action` 计数器遍历动作列表，每次+1
2. **验证是安全网**：LLM生成的代码不一定对，`ast.parse()` 是最基本的检查
3. **三路分支是新模式**：之前最多二路（继续/结束），这里有三路（继续/完成/出错）
4. **简化了浏览器控制**：原始notebook用Playwright操作真实浏览器+DOM，我们聚焦于代码生成循环的模式
