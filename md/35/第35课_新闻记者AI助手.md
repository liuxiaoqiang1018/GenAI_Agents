# 第35课：新闻记者 AI 助手（Journalism-Focused AI Assistant）

## 课程概述

本课构建一个**面向新闻记者的多功能文章分析系统**：用户输入意图后，LLM 自动识别需要执行的分析类型，路由到 5 个专业模块——摘要、事实核查、语气分析、引语提取、语法偏见审查。支持单项和"全部报告"模式。

**核心模式**：意图分类 + 多路条件路由 + 5 模块并行分析 + Markdown 报告生成

## 架构流程

```
用户输入 + 文章 → [意图分类] LLM识别用户要做什么
                       ↓
              actions: ["summarization", "fact-checking", ...]
                       ↓
              [多路条件路由]
              ↙    ↙    ↓    ↘    ↘
          摘要  事实核查  语气  引语  语法偏见
              ↘    ↘    ↓    ↙    ↙
                     结束 → 格式化Markdown报告
```

## 五大分析模块

| 模块 | 功能 | Prompt 关键指令 |
|---|---|---|
| **摘要** | 150-200字新闻摘要，关注关键人物和数据 | Map-Reduce 分块摘要再合并 |
| **事实核查** | 逐条检查事实准确性，标记 confirmed/refuted/vague | 结构化 JSON 输出 |
| **语气分析** | 判断文章是中立/正面/批判/有立场 | 用原文举例支撑 |
| **引语提取** | 识别直接引用，标注说话人和上下文 | 无引用则返回"无" |
| **语法偏见审查** | 检查语法拼写标点和偏见倾向 | 列出问题和改进建议 |

## 关键概念标注

| 概念 | 在代码中的位置 | Java 类比 |
|---|---|---|
| **意图分类** | `categorize_user_input()` — LLM 结构化输出 actions 列表 | 请求路由网关 |
| **多路条件路由** | `add_conditional_edges()` — actions 映射到不同节点 | 策略模式 + 路由表 |
| **Map-Reduce 摘要** | 分块摘要 → 合并 | Fork/Join |
| **结构化输出** | `FactCheckResult(TypedDict)` + `with_structured_output` | POJO 映射 |
| **文本分块** | `chunk_large_text()` — 处理超长文章 | 分页处理 |

## 本课特点

1. **5 路条件路由**：一个分类节点分发到 5 个分析模块，路由最多的一课
2. **意图识别驱动**：用户说"摘要"就只跑摘要，说"全部报告"就跑全部
3. **Map-Reduce 摘要**：长文章分块摘要再合并，适用于超长文本
4. **事实核查 + 搜索**：LLM 标记可疑声明，建议搜索关键词验证
5. **Markdown 报告生成**：最终输出完整分析报告并保存文件
