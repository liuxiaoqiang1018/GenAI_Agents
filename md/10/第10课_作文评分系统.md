# 第10课 - 作文评分系统（条件提前终止）

## 环境准备

```bash
pip install langgraph langchain-core httpx python-dotenv
```

`main_debug.py` 只需要 `httpx` 和 `python-dotenv`。

## 概述

本课构建一个**自动作文评分系统**，按4个维度逐步评分：
1. **内容相关性**（Relevance）：作文是否切题，权重 30%
2. **语法检查**（Grammar）：语言表达和语法质量，权重 20%
3. **结构分析**（Structure）：组织架构和逻辑流畅度，权重 20%
4. **深度分析**（Depth）：批判性思维和见解深度，权重 30%

核心特点：**条件提前终止** —— 如果前面的评分太低，直接跳过后续检查，提前出分。

## 动机

传统评分系统会对每篇作文做完整评估，浪费资源。实际场景中：
- 跑题的作文 → 不用检查语法了
- 语法太差 → 不用分析结构了
- 结构混乱 → 不用评深度了

这就是**条件提前终止**模式：每一步设门槛，不达标就短路到最终评分。

```
   └──────────────┘
```

## 核心概念

### 1. 条件提前终止（Conditional Early Termination）

每个评分节点后面跟一个条件判断，不达标就跳过后续步骤：

```python
# 相关性 < 0.5 → 跳过语法、结构、深度，直接算总分
workflow.add_conditional_edges(
    "check_relevance",
    lambda x: "check_grammar" if x["relevance_score"] > 0.5 else "calculate_final_score"
)
# 语法 < 0.6 → 跳过结构、深度
# 结构 < 0.7 → 跳过深度
```

**Java 类比**：
```java
// 像 Servlet Filter 链的短路逻辑
public void doFilter(Request req, Response resp, FilterChain chain) {
    if (relevanceScore <= 0.5) {
        // 短路：不再往后传，直接返回
        calculateFinalScore(state);
        return;
    }
    chain.doFilter(req, resp);  // 继续下一个 Filter
}
```

### 2. 加权评分（Weighted Scoring）

```python
final_score = (
    relevance_score * 0.3 +   # 相关性 30%
    grammar_score   * 0.2 +   # 语法 20%
    structure_score * 0.2 +   # 结构 20%
    depth_score     * 0.3     # 深度 30%
)
```

如果提前终止，未评分的维度保持 0 分，总分自然会很低。这个设计既合理又简洁。

### 3. 评分提取（Score Extraction）

LLM 返回的是自然语言文本，需要用正则从中提取数字：

```python
def extract_score(content: str) -> float:
    match = re.search(r'Score:\s*(\d+(\.\d+)?)', content)
    if match:
        return float(match.group(1))
```

通过 prompt 约束 LLM 以 `Score: 0.85` 格式开头，再用正则提取。这是结构化输出的简单方案。

## 关键洞察

1. **提前终止节省资源**：跑题的作文只需1次LLM调用（相关性），不用再调3次
2. **门槛递增**：相关性>0.5、语法>0.6、结构>0.7 —— 越往后要求越高，像逐轮淘汰
3. **未评维度=零分**：提前终止时未评分的项是0，通过加权自然得到低分，不需要额外逻辑
4. **正则提取是过渡方案**：实际生产中会用结构化输出（JSON mode）替代正则匹配
