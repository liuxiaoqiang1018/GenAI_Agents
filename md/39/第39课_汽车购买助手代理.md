# 第39课：汽车购买助手代理（Smart Car Buyer Agent）

## 课程概述

本课构建一个**交互式汽车购买助手**：通过多轮对话了解用户需求 → LLM 构建搜索过滤条件 → 爬取汽车网站列表 → 展示匹配车辆 → 用户选择后获取详细信息和网络口碑 → 支持继续筛选或结束。

**核心模式**：多轮对话需求收集 + LLM 构建搜索参数 + 网站爬虫 + 条件循环（选车/改条件/结束）

## 架构流程

```
[询问需求] ←──────────────────────────┐
    ↓                                  │
[条件路由]                             │
  ├─ 信息不足 → 继续询问 ──────────→ ┘
  ├─ 无关问题 → 结束
  └─ 信息充足 → [构建过滤条件]
                    ↓
              [搜索车辆列表] ← 爬取网站
                    ↓
              展示匹配结果，用户选择
                    ↓
              [条件路由]
              ├─ 选择某辆 → [获取详细信息] + DuckDuckGo搜索口碑
              ├─ 修改条件 → 返回[询问需求]
              └─ 结束对话 → END
```

## 关键概念标注

| 概念 | 在代码中的位置 | Java 类比 |
|---|---|---|
| **多轮需求收集** | `ask_user_needs()` — LLM 引导+总结用户需求 | 表单向导 |
| **结构化输出分类** | `UserNeeds(BaseModel)` + `NextStep(Enum)` | DTO + 枚举路由 |
| **LLM 构建搜索参数** | `build_filters()` — 根据需求生成 URL 过滤参数 | 智能查询构建器 |
| **网站爬虫** | `AutotraderInterface` — Playwright 无头浏览器 | Selenium / Jsoup |
| **用户响应分类** | `UserResponse(BaseModel)` — select/refine/end 三选一 | 枚举路由 |
| **外部搜索增强** | `DuckDuckGoSearchResults` — 搜索车型口碑 | 搜索引擎集成 |
| **MessagesState** | 对话历史自动累积 | 对话上下文 |

## 本课特点

1. **真正的交互式 Agent**：不是一次性处理，而是多轮对话 + 用户选择 + 循环
2. **LLM 构建搜索参数**：用户说"预算20万，要SUV"，LLM 自动转换为网站 URL 参数
3. **Playwright 爬虫**：无头浏览器爬取真实汽车网站（原教程爬 Autotrader）
4. **三路循环**：选车→看详情 / 改条件→重搜 / 结束对话
5. **多源信息整合**：网站列表 + 详情页 + DuckDuckGo 口碑搜索
