# 第19课 - 多平台内容生成代理（线性+并行+汇聚）

## 环境准备

```bash
pip install langgraph langchain-core httpx python-dotenv
```

`main_debug.py` 只需要 `httpx` 和 `python-dotenv`。

## 概述

本课构建一个**多平台内容生成代理**，将一篇文章自动转化为多个社交平台的内容：

1. **文本摘要** → 提取核心要点
2. **内容调研** → 补充相关背景知识
3. **意图匹配** → 确定要生成哪些平台的内容
4. **平台内容生成**（并行）→ 同时为微博、公众号、小红书、知乎生成内容
5. **内容合并** → 汇总所有平台内容

核心模式：**线性前处理 + 并行分发 + 汇聚合并**，综合了前几课的模式。

## 核心概念

### 1. 线性+并行+汇聚（Pipeline + Fan-out + Merge）

这是前几课模式的组合：
- **线性流水线**（第4课）：摘要→调研→意图匹配
- **并行分发**（第14课 Send）：4个平台同时生成
- **汇聚合并**（第17课菱形）：所有平台结果合并

**Java 类比**：
```java
// Spring Batch: 顺序处理 → 并行分区 → 汇聚
@Bean
public Job contentJob() {
    return jobBuilder
        .start(summaryStep)           // 线性
        .next(researchStep)           // 线性
        .next(intentStep)             // 线性
        .next(platformPartitioner)    // 并行分区
        .next(mergeStep)              // 汇聚
        .build();
}
```

### 2. 平台适配策略

同一份摘要+调研，用**不同的 system prompt** 适配不同平台：
- **微博**：140字以内，带话题标签
- **公众号**：长文，标题+正文+总结
- **小红书**：种草风格，emoji丰富
- **知乎**：深度分析，专业口吻

### 3. 条件跳过（Conditional Skip）

如果用户没选某平台，该节点返回空内容：
```python
def weibo_node(state):
    if "微博" not in state["platforms"]:
        return {"contents": [""]}  # 跳过
    ...
```

### 4. 与前几课的对比

| 维度 | 第14课（合同审查） | 第17课（TTS） | **第19课（内容生成）** |
|------|-------------------|-------------|---------------------|
| 前处理 | 分类+检索 | 分类 | **摘要+调研+意图匹配** |
| 并行模式 | Send动态扇出 | 4选1分支 | **4路固定并行** |
| 汇聚方式 | operator.add | 菱形到TTS | **合并到最终输出** |
| 结构 | 线性→并行→线性 | 分支→汇聚 | **线性→并行→汇聚** |

## 关键洞察

1. **这是前几课的"毕业作品"**：线性流水线+并行Send+汇聚合并，一次性用到了多个模式
2. **平台适配的核心是prompt**：同样的内容，换不同风格的system prompt就能适配不同平台
3. **条件跳过很实用**：不是所有平台都需要生成，跳过逻辑避免浪费LLM调用
4. **operator.add自动合并**：4个并行节点各自返回 `[content]`，自动拼接成完整列表
