```bash
pip install httpx python-dotenv
```

## 概述

本课构建一个**销售通话分析器**，分析销售对话并生成结构化报告：

1. **转录**（Transcribe）→ 音频转文字（本课用示例文本代替）
2. **分析**（Analyze）→ Agent 对通话内容进行多维度分析
3. **报告**（Report）→ 生成结构化JSON报告

分析维度：
- **情感分析**：客户情绪变化
- **关键短语**：核心需求和关注点
- **痛点识别**：客户的问题和不满
- **销售效果**：销售人员的表现评估
- **改进建议**：具体的优化方向

核心模式：**CrewAI 的 Agent/Task/Crew 模式** —— 定义Agent（谁来做）、Task（做什么）、Crew（怎么组织执行）。

## 核心概念

### 1. CrewAI 的 Agent/Task/Crew 模式

CrewAI 把AI工作抽象为三个概念：

```python
# Agent: 谁来做（角色定义）
agent = Agent(
    role="销售通话分析师",
    goal="从通话中提供可执行的洞察",
    backstory="你是资深销售培训专家..."
)

# Task: 做什么（任务定义）
task = Task(
    description="分析这段通话：情感、关键词、痛点、效果、建议",
    agent=agent
)

# Crew: 执行（组织和运行）
crew = Crew(agents=[agent], tasks=[task])
result = crew.kickoff()
```

**Java 类比**：
```java
// 像 Spring Batch: Job(Task) + ItemProcessor(Agent) + Launcher(Crew)
@Bean public Job analysisJob() {
    return jobBuilder
        .start(new StepBuilder("analyze")
            .processor(new CallAnalysisAgent())  // Agent
            .build())
        .build();
}
launcher.run(job, params);  // Crew.kickoff()
```

### 2. 与前几课Agent模式的对比

| 维度 | 第25课（工具调用Agent） | 第27课（群聊） | **第28课（CrewAI）** |
|------|----------------------|-------------|-------------------|
| Agent定义 | 隐式（prompt里） | system prompt | **显式（role/goal/backstory）** |
| 任务定义 | 用户直接输入 | 群聊讨论 | **Task对象（结构化）** |
| 执行方式 | Agent循环 | Manager调度 | **Crew执行** |
| 框架 | 手工实现 | 手工实现 | **CrewAI模式** |

## 关键洞察

1. **Agent/Task/Crew 是组织模式**：不是新技术，而是一种清晰的职责划分方式
2. **role+goal+backstory = 更好的角色扮演**：比单纯的system prompt更结构化
3. **Task的description是核心**：详细的任务描述决定分析质量
4. **结构化输出很重要**：JSON格式的报告比自由文本更容易后续处理
