# 第32课：记忆增强邮件代理（Memory-Enhanced Email Agent）

## 课程概述

本课构建一个**拥有三层记忆的智能邮件助手**：能自动分类邮件（忽略/通知/回复）、利用历史案例（情景记忆）辅助决策、记住用户偏好（语义记忆）、根据反馈自我改进提示词（程序记忆）。

**核心模式**：三层记忆架构（情景 + 语义 + 程序）+ 邮件分类路由 + Prompt 自我优化

## 三层记忆架构

| 记忆类型 | 作用 | 类比 | Java 类比 |
|---|---|---|---|
| **情景记忆（Episodic）** | 存储历史邮件分类案例，作为 Few-Shot 示例 | "上次类似的邮件我是这样处理的" | 案例库 / Redis 缓存 |
| **语义记忆（Semantic）** | 存储用户偏好和长期知识 | "这个用户喜欢简洁回复" | 用户画像数据库 |
| **程序记忆（Procedural）** | 存储和更新系统提示词（Prompt） | "我学会了更好的分类方式" | 配置中心 / 热更新 |

## 架构流程

```
收到邮件 → [邮件分类（情景记忆辅助）]
  ├─ ignore（垃圾邮件）→ 结束
  ├─ notify（仅通知）→ 结束
  └─ respond（需回复）→ ReAct Agent（语义记忆 + 工具）
                          ├─ 搜索记忆（用户偏好）
                          ├─ 查日历
                          ├─ 写邮件
                          └─ 管理记忆
                              ↓
                            生成回复 → 结束

[反馈循环] 用户反馈 → Prompt 优化器 → 更新程序记忆（提示词）
```

## 关键概念标注

| 概念 | 在代码中的位置 | 说明 |
|---|---|---|
| **InMemoryStore** | LangGraph 内置的键值存储 | 三种记忆都存在这里 |
| **Few-Shot 示例** | `format_few_shot_examples()` | 从情景记忆中取出历史案例拼进 Prompt |
| **结构化输出** | `Router(BaseModel)` + `with_structured_output` | LLM 输出固定格式（分类+推理） |
| **LangMem 工具** | `create_manage_memory_tool` / `create_search_memory_tool` | Agent 可以自主读写记忆 |
| **ReAct Agent** | `create_react_agent()` | 预构建的工具调用 Agent |
| **Prompt 优化器** | `create_multi_prompt_optimizer` | 根据反馈自动改进提示词 |
| **Namespace** | `("email_assistant", user_id, "examples")` | 按用户隔离记忆空间 |

## 本课特点（对比之前课程）

1. **三层记忆架构**：第22课只有短期+长期两层，本课是情景+语义+程序三层
2. **记忆驱动分类**：Few-Shot 示例从记忆中动态检索，不是硬编码
3. **Prompt 自我优化**：程序记忆让 Agent 能"学会"更好的工作方式
4. **工具型记忆**：Agent 通过工具接口读写记忆，而非框架自动管理
5. **LangMem 库**：第一次使用 LangMem，专门做 Agent 记忆管理
