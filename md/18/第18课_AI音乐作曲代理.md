# 第18课 - AI音乐作曲代理（逐步丰富的提示词链）

## 环境准备

```bash
pip install langgraph langchain-core httpx python-dotenv music21
```

`main_debug.py` 需要 `httpx`、`python-dotenv` 和 `music21`。

生成的 `.mid` 文件可以双击用 Windows 媒体播放器直接播放。`music21` 是免费开源库，不需要 API 密钥。

## 概述

本课构建一个**AI音乐作曲代理**，从用户描述自动生成音乐作品：

1. **旋律生成** → 根据用户输入生成旋律
2. **和声创作** → 基于旋律创建和声
3. **节奏分析** → 基于旋律+和声建议节奏
4. **风格适配** → 基于旋律+和声+节奏适配到指定风格
5. **MIDI转换** → 转为可播放的MIDI文件（模拟）

核心模式：**逐步丰富（Progressive Enrichment）** —— 每一步在前面的基础上增加一个维度。

## 架构

```
用户输入："创作一首欢快的C大调钢琴曲"
   │
   ▼
┌──────────┐
│ 旋律生成  │  → 音符序列
└──────────┘
   │ + 旋律
   ▼
┌──────────┐
│ 和声创作  │  → 和弦进行
└──────────┘
   │ + 旋律 + 和声
   ▼
┌──────────┐
│ 节奏分析  │  → 节拍模式
└──────────┘
   │ + 旋律 + 和声 + 节奏
   ▼
┌──────────┐
│ 风格适配  │  → 完整乐谱
└──────────┘
   │
   ▼
┌──────────┐
│ MIDI转换  │  → 可播放文件
└──────────┘
```

## 核心概念

### 1. 逐步丰富模式（Progressive Enrichment）

与第16课的区别：第16课是"上一步的输出全部传给下一步"，本课是**每步增加一个新维度**：

```
step1: melody = generate(user_input)
step2: harmony = generate(melody)           # 基于旋律
step3: rhythm = generate(melody, harmony)   # 基于旋律+和声
step4: composition = generate(melody, harmony, rhythm, style)  # 全部维度
```

**Java 类比**：像装饰器模式层层包装
```java
Music result = new StyleAdapter(style,
    new RhythmLayer(
        new HarmonyLayer(
            new MelodyGenerator(userInput)
        )
    )
).compose();
```

### 2. 与第16课的对比

| 维度 | 第16课（GIF生成） | **第18课（音乐作曲）** |
|------|------------------|---------------------|
| 传递方式 | 角色描述传到所有步骤 | **每步累加一个维度** |
| 上下文增长 | 固定（角色描述不变） | **递增（旋律→旋律+和声→...）** |
| 最终步 | 图片→GIF合成 | **乐谱→MIDI转换** |
| 领域 | 视觉创作 | **音乐创作** |

## 关键洞察

1. **逐步丰富 ≠ 简单传递**：每步的 prompt 包含越来越多的上下文，像滚雪球
2. **领域知识在 prompt 里**：旋律用音符表示、和声用和弦表示——LLM 充当领域专家
3. **模式可复用**：把"旋律/和声/节奏"换成"需求/设计/实现"，就是软件开发流水线
