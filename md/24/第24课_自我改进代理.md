# 第24课 - 自我改进代理（反思-学习循环）

## 环境准备

```bash
pip install httpx python-dotenv
```

本课不使用 LangGraph，聚焦于**自我改进机制**。

## 概述

本课构建一个**自我改进Agent**，能够通过反思对话来提升自己的表现：

1. **对话**（Respond）→ 正常回答用户问题
2. **反思**（Reflect）→ 回顾对话历史，生成改进洞察
3. **学习**（Learn）→ 从洞察中提取关键要点，更新自身行为

循环：对话 → 对话 → ... → **反思+学习** → 改进后的对话 → ...

核心模式：**反思-学习循环** —— Agent 定期审视自己的表现，提取改进建议，注入到后续对话中。

## 架构

```
用户输入
   │
   ▼
┌──────────────────────────────────────┐
│            Prompt 组装                │
│  System: "你是自我改进的AI助手"        │
│  History: [对话历史]                  │
│  Human: {用户输入}                    │
│  System: "改进洞察: {insights}"  ← 学习成果注入
│                                      │
└──────────────────────────────────────┘
   │
   ▼
 LLM → 回复用户
   │
   └──→ 追加到对话历史

（每隔N轮触发）
   │
   ▼
┌──────────┐     ┌──────────┐
│  反思     │ ──→ │  学习     │
│ (Reflect) │     │ (Learn)  │
└──────────┘     └──────────┘
   │                  │
   │ 分析对话历史      │ 提取关键要点
   │ 生成改进洞察      │ 更新 insights
   │                  │
   └──────────────────┘
          │
          ▼
      insights 注入到后续对话的 prompt 中
```

## 核心概念

### 1. 反思（Reflect）

Agent 回顾自己的对话历史，生成改进建议：

```python
def reflect(conversation_history):
    prompt = "回顾以下对话历史，分析哪些回答可以改进，给出具体建议"
    insights = call_llm(prompt + conversation_history)
    return insights
```

### 2. 学习（Learn）

从反思洞察中提取可执行的关键要点：

```python
def learn(insights):
    prompt = "从以下洞察中提取关键行为要点，用于改进未来的回答"
    learned = call_llm(prompt + insights)
    return learned  # 例如："回答要更简洁"、"多举例子"
```

### 3. 洞察注入（Insight Injection）

学习成果注入到后续对话的 system prompt 中：

```python
# 改进前的prompt
system = "你是AI助手"
# 改进后的prompt
system = "你是AI助手。改进洞察：回答要更简洁，多举例子，主动追问"
```

**Java 类比**：
```java
// 像AOP的后置处理器，每次交互后触发反思
@AfterReturning("execution(* respond(..))")
public void afterResponse() {
    if (interactionCount % N == 0) {
        String insights = reflectionService.reflect(history);
        String learned = learningService.learn(insights);
        promptTemplate.updateInsights(learned);  // 注入改进
    }
}
```

### 4. 与前几课的对比

| 维度 | 第13课（项目管理自反思） | 第22课（记忆增强） | **第24课（自我改进）** |
|------|----------------------|-------------------|---------------------|
| 反思对象 | 项目计划的风险 | 无反思 | **自身对话质量** |
| 改进方式 | 重新排期分配 | 存事实 | **更新行为洞察** |
| 反思频率 | 每轮自动 | 无 | **手动/定期触发** |
| 目标 | 降低项目风险 | 记住事实 | **提升回答质量** |

## 关键洞察

1. **反思 = 让LLM审查自己的输出**：用一个LLM调用分析之前对话的质量
2. **学习 = 把反思结果精炼后注入prompt**：不是存原始对话，而是提取"行为准则"
3. **和第13课的区别**：第13课反思的是"任务产出"（项目计划好不好），本课反思的是"Agent自身行为"（回答好不好）
4. **和第22课的关系**：第22课的长期记忆存"事实"，本课存"行为改进建议"——两者可以结合
