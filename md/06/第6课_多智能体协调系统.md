python -m venv venv
venv\Scripts\activate  # Windows

`main_debug.py` 不需要 LangGraph，只需要 `openai` 和 `python-dotenv`。

## 概述

ATLAS（Academic Task Learning Agent System）是一个**多智能体协调系统**，多个专业 Agent 协作帮助学生管理学习任务。

与第4课（单条流水线）不同，本课的图结构包含：
- **条件路由**：根据分析结果决定激活哪些 Agent
- **并行执行**：多个 Agent 可以同时工作
- **协调器模式**：一个 Coordinator 负责调度

## 动机

现实中的 AI 应用很少只有一个 Agent。复杂任务需要多个专业 Agent 协作：
- 一个负责排日程（计划者）
- 一个负责整理笔记（笔记员）
- 一个负责提供建议（顾问）
- 一个负责协调它们（协调器）

这就是**多智能体系统**的核心思想。

## 架构

```
用户请求
   │
   ▼
┌──────────┐
│  协调器   │  分析请求，决定需要哪些 Agent
└──────────┘
   │
   ▼
┌──────────┐
│ 档案分析  │  分析学生信息和学习偏好
└──────────┘
   │
   ├─── 条件路由（根据分析结果）
   │
   ▼           ▼           ▼
┌──────┐  ┌──────┐  ┌──────┐
│计划者 │  │笔记员 │  │ 顾问  │   ← 可并行执行
└──────┘  └──────┘  └──────┘
   │           │           │
   └───────────┼───────────┘
               ▼
          ┌──────────┐
          │  执行汇总  │  汇总所有 Agent 的结果
          └──────────┘
               │
               ▼
              结束
```

## 核心概念

### 1. Coordinator（协调器）
分析用户请求，决定需要调用哪些 Agent。

```python
# 协调器分析后返回
{"required_agents": ["PLANNER", "NOTEWRITER"], "priority": {...}}
```

**Java 类比**：像 Spring Cloud Gateway 的路由器，根据请求内容分发到不同微服务。

### 2. ReAct Agent（推理-行动代理）
每个 Agent 都继承自 ReActAgent 基类，遵循 ReAct 模式：
- **Thought**（思考）：分析当前情况
- **Action**（行动）：选择工具执行
- **Observation**（观察）：查看执行结果
- **Response**（响应）：生成最终回答

**Java 类比**：像模板方法模式（Template Method Pattern），基类定义流程骨架，子类实现具体步骤。

### 3. Conditional Edges（条件路由）
根据协调器的分析结果，动态决定走哪些分支。

```python
workflow.add_conditional_edges(
    "profile_analyzer",        # 从这个节点出发
    route_to_parallel_agents,  # 路由函数
    ["planner", "notewriter", "advisor"]  # 可能的目标
)
```

**Java 类比**：像 `switch-case` + 策略模式，根据条件选择不同处理链。

### 4. Parallel Execution（并行执行）
LangGraph 支持同时执行多个分支。当条件路由返回多个目标时，它们并行运行。

**Java 类比**：像 `CompletableFuture.allOf()` 等待多个异步任务全部完成。

### 5. State Reducer（状态合并）
多个 Agent 并行执行后，结果需要合并到同一个 State。使用 `Annotated[Dict, dict_reducer]` 来定义合并策略。

```python
class AcademicState(TypedDict):
    messages: Annotated[List, add]           # 列表追加
    results: Annotated[Dict, dict_reducer]   # 字典深度合并
```

**Java 类比**：像 `Stream.reduce()` 操作。

## 本课的三种 Agent

| Agent | 职责 | 输入 | 输出 |
|-------|------|------|------|
| 计划者（Planner） | 分析日程和任务，生成学习计划 | 日历 + 任务列表 | 时间安排方案 |
| 笔记员（NoteWriter） | 根据学习风格生成学习笔记 | 学习偏好 + 课程信息 | 结构化笔记 |
| 顾问（Advisor） | 提供个性化学习建议 | 学生档案 + 当前困难 | 改进建议 |

## 与前几课的对比

| 维度 | 第4课（LangGraph 线性） | 本课（多智能体） |
|------|------------------------|----------------|
| 图结构 | 线性（A→B→C） | **有分支、条件路由、并行** |
| Agent 数量 | 0（纯函数节点） | **3个 + 1个协调器** |
| 路由方式 | 固定顺序 | **动态条件路由** |
| LLM 调用 | 固定3次 | **动态，取决于激活了哪些 Agent** |
| 状态管理 | 简单 TypedDict | **带 Reducer 的复杂状态** |

## 关键洞察

1. **协调器是多智能体系统的核心**：它不做具体工作，只负责分析和调度
2. **ReAct 模式让 Agent 有推理能力**：不是直接生成结果，而是先思考、再行动、再观察
3. **条件路由 + 并行执行 = 灵活的工作流**：不是所有 Agent 都需要参与每次请求
4. **Subgraph 让复杂系统可管理**：每个 Agent 内部有自己的子图（分析→生成）
