# 第3课：简单数据分析代理（Data Analysis Agent）

**本教程基于 `simple_data_analysis_agent_notebook-pydanticai.ipynb`，演示如何让 AI 代理用自然语言查询数据。**

## 概述

这节课的核心：**让 Agent 自己写 Pandas 查询代码来分析数据**。

用户用自然语言提问（比如"平均价格是多少"），Agent 自动生成 `df['Price'].mean()` 这样的查询语句，执行后返回结果。

## 新概念

### 1. 依赖注入（Dependency Injection）

PydanticAI 用 `Deps` 类把数据传给工具，工具通过 `RunContext` 访问。

```
Java 类比：
    @dataclass Deps     →  相当于 Spring 的 @Component / @Bean
    RunContext[Deps]     →  相当于 Spring 的 @Autowired 注入
    ctx.deps.df          →  相当于 this.dataService.getDataFrame()
```

### 2. 自定义工具（Custom Tool）

PydanticAI 没有内置的 DataFrame 工具，所以我们自己写一个：

- Agent（LLM）决定要执行什么查询
- 工具函数用 `pd.eval()` 执行查询
- 结果返回给 Agent 生成自然语言回答

### 3. 重试机制（ModelRetry）

LLM 生成的查询代码可能有语法错误（比如列名大小写搞错）。`ModelRetry` 让 Agent 在出错时自动重试，最多 10 次。

```
Java 类比：
    ModelRetry  →  相当于 Spring Retry 的 @Retryable
    retries=10  →  相当于 maxAttempts=10

    执行过程类似：
    try {
        pd.eval(query);      // 执行 Agent 生成的查询
    } catch (Exception e) {
        throw new RetryableException("请换个写法重试");  // 告诉 Agent 换种方式
    }
```

## 核心流程

```
用户提问："平均价格是多少？"
    │
    ▼
┌──────────────────────────────────────────┐
│ Agent 拼装 Prompt（包含工具描述）发给 LLM   │
│                                          │
│ system: "你是AI助手，帮助查询DataFrame..."  │
│ tools: [df_query: 用pd.eval执行查询]       │
│ user: "平均价格是多少？"                    │
└──────────────────────────────────────────┘
    │
    ▼
┌──────────────────────────────────────────┐
│ LLM 决策：需要调用 df_query 工具           │
│ 返回：df_query(query="df['Price'].mean()") │
└──────────────────────────────────────────┘
    │
    ▼
┌──────────────────────────────────────────┐
│ 工具执行：pd.eval("df['Price'].mean()")   │
│ → 如果出错，抛 ModelRetry，LLM 换种写法重试 │
│ → 成功返回：51145.36                      │
└──────────────────────────────────────────┘
    │
    ▼
┌──────────────────────────────────────────┐
│ Agent 把工具结果拼回 Prompt 发给 LLM       │
│ LLM 生成自然语言回答：                     │
│ "汽车的平均售价约为 $51,145.36"            │
└──────────────────────────────────────────┘
```

**关键点**：LLM 生成的查询可能出错（比如写成 `df['price']` 小写 p），工具捕获异常后通过 `ModelRetry` 告诉 LLM "你写错了，原因是xxx"，LLM 会换种写法重试。

## 实现

### 安装依赖

```bash
pip install pydantic-ai-slim[openai] pandas numpy python-dotenv
```

### 生成示例数据

```python
n_rows = 1000
# 生成汽车销售数据：日期、品牌、车型、颜色、年份、价格、里程、排量、油耗、销售员
df = pd.DataFrame({
    'Date': dates,
    'Make': np.random.choice(makes, n_rows),
    'Price': np.random.uniform(20000, 80000, n_rows).round(2),
    # ...
})
```

### 定义依赖

```python
@dataclass
class Deps:
    df: pd.DataFrame   # 把 DataFrame 作为依赖注入给工具
```

### 创建 Agent 和工具

```python
agent = Agent(
    model='openai:gpt-4o-mini',
    system_prompt="你是AI助手，帮助从DataFrame中提取信息...",
    deps_type=Deps,
    retries=10,          # 允许重试 10 次
)

@agent.tool
async def df_query(ctx: RunContext[Deps], query: str) -> str:
    """用 pd.eval 执行查询"""
    try:
        return str(pd.eval(query, target=ctx.deps.df))
    except Exception as e:
        raise ModelRetry(f'查询无效: {e}')   # 出错时让 Agent 重试
```

### 使用

```python
deps = Deps(df=df)
response = agent.run_sync("平均价格是多少？", deps=deps)
```

## 总结

这节课的价值：

- **工具调用的完整闭环**：LLM 生成代码 → 执行 → 可能出错 → 重试 → 最终成功
- **依赖注入**：优雅地把数据传递给工具
- **错误恢复**：Agent 不是一次性的，它能从错误中学习并调整

这比第1课的工具调用复杂在哪里？第1课的工具输入是确定的（距离数字），这课的工具输入是 LLM **自己生成的代码**，所以需要重试机制来处理 LLM 的"手误"。
