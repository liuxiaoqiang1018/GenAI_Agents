# 第21课 - 侦探推理游戏（子图 + 多角色交互 + 游戏循环）

## 环境准备

```bash
pip install langgraph langchain-core httpx python-dotenv
```

`main_debug.py` 只需要 `httpx` 和 `python-dotenv`。

## 概述

本课构建一个**侦探推理游戏**，玩家扮演侦探，审问嫌疑人，找出凶手：

1. **角色生成** → LLM 生成嫌疑人角色（含背景故事和动机）
2. **故事生成** → LLM 创建谋杀案场景（地点、凶器、时间线）
3. **叙述者开场** → 介绍案件背景
4. **审问循环**（子图）→ 选择嫌疑人 → 提问 → 角色回答（多轮对话）
5. **猜测凶手** → 3次机会猜凶手

核心模式：**子图（Sub-Graph）** —— 审问对话是一个独立的 LangGraph 图，被嵌入到外层游戏循环图中。

## 动机

复杂的交互系统需要**模块化**：
- 外层：游戏主循环（生成→叙述→审问→猜测）
- 内层：对话子循环（介绍→提问→回答→继续/结束）

把对话逻辑封装成子图，外层图不用关心对话内部细节，就像函数封装一样。

## 核心概念

### 1. 子图（Sub-Graph）

一个 LangGraph 图作为另一个图的节点：

```python
# 内层子图：对话循环
conversation_graph = StateGraph(ConversationState)
conversation_graph.add_node("intro", intro_node)
conversation_graph.add_node("ask", ask_node)
conversation_graph.add_node("answer", answer_node)
conversation_compiled = conversation_graph.compile()

# 外层图：把子图作为一个节点
game_graph = StateGraph(GameState)
game_graph.add_node("conversation", conversation_compiled)  # ← 子图嵌入
```

**Java 类比**：
```java
// 像 Spring 的嵌套事务 / 组合模式
@Service
public class GameService {
    @Autowired
    private ConversationService conversationService;  // 独立的子模块

    public void playGame() {
        createCharacters();
        createStory();
        while (guessesLeft > 0) {
            // 调用子模块（对应子图）
            conversationService.startConversation(character);
            if (guess()) break;
        }
    }
}
```

### 2. 多角色人格（Multi-Persona）

每个嫌疑人有独立的人格设定，审问时 system prompt 不同：

```python
def answer_node(state):
    system = f"你是{character.name}，{character.backstory}。"
    if character.is_killer:
        system += "你是凶手但要隐藏这个事实。"
    response = call_llm(question, system)
```

### 3. 游戏状态管理

外层和内层各自维护不同的状态：
- **外层 GameState**：角色列表、故事、猜测次数、当前角色
- **内层 ConversationState**：对话历史、当前角色、消息列表

## 关键洞察

1. **子图 = 函数封装**：把复杂的对话逻辑封装成子图，外层只需调用一个节点
2. **多角色 = 多个 system prompt**：同一个 answer 函数，传入不同角色的背景，就产生不同人格的回答
3. **凶手会说谎**：凶手角色的 prompt 里有"隐藏真相"的指令，这是对抗式 prompt 的简单应用
4. **状态隔离**：子图有自己的状态，不会污染外层状态——像函数有自己的局部变量
