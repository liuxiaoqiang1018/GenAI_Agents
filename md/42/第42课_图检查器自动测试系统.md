# 第42课：图检查器自动测试系统（Graph Inspector System）

## 课程概述

本课构建一个**用 Agent 测试 Agent 的元编程系统**：输入一个 LangGraph 图，系统自动分析图结构 → 为每个节点生成描述 → 创建测试人设（功能测试/注入攻击/漏洞猎手） → 生成测试用例 → 自动执行测试 → 分析结果判定通过/失败。

**核心模式**：图结构分析 + Send 并行测试 + 测试人设生成 + 子图嵌套执行 + 结果断言

## 架构流程

```
输入目标图 → [静态分析] 编译图、提取节点/边/工具信息(NetworkX)
               ↓
          [生成节点描述] LLM为每个节点生成功能描述
               ↓
          [生成测试人设] LLM创建3种测试角色
               ↓
          [生成测试用例] 每个(节点,测试者)组合生成3+测试用例
               ↓ (条件循环: 还有组合？)
          [执行测试] 子图: 生成输入→运行图→收集结果
               ↓
          [分析结果] LLM判断是否通过验收标准
               ↓ (条件循环: 还有结果？)
          输出测试报告 → 结束
```

## 关键概念标注

| 概念 | 在代码中的位置 | Java 类比 |
|---|---|---|
| **静态分析** | `static_test()` — 编译图，用 NetworkX 提取节点/边/工具 | 反射 + AST 分析 |
| **节点描述生成** | `generate_node_descriptions()` — LLM 根据输入输出推断节点功能 | JavaDoc 生成 |
| **测试人设** | `generate_testers()` — 功能测试者/注入攻击者/漏洞猎手 | 测试角色 |
| **测试用例生成** | `generate_test_cases()` — 每个节点×每个测试者生成多个测试 | 参数化测试 |
| **子图执行** | `sub_builder.compile()` 作为节点执行测试 | 测试运行器 |
| **结果断言** | `analize_results()` — LLM 根据验收标准判断通过/失败 | Assert |
| **条件循环** | `more_test_cases` / `more_results` — 处理完所有才结束 | for 循环 |

## 本课特点

1. **元编程**：用 Agent 测试 Agent——输入一个 LangGraph 图，输出测试报告
2. **自动生成测试人设**：LLM 创建不同视角的测试者（安全/功能/边界）
3. **NetworkX 图分析**：静态分析图结构，提取节点类型、工具、边关系
4. **子图嵌套执行测试**：测试执行本身是一个子图（生成输入→执行→收集）
5. **安全测试**：包含注入攻击和越狱测试的测试人设
