# 第34课：AI 周报生成代理（AInsight）

## 课程概述

本课构建一个 **AI/ML 新闻周报自动生成系统**：三个专业 Agent 分工协作——搜索者检索最新新闻、摘要者逐篇生成通俗摘要、发布者编排成 Markdown 报告并保存文件。

**核心模式**：线性三步流水线（搜索→摘要→发布）+ 三 Agent 分工

## 架构流程

```
搜索者 Agent → 摘要者 Agent → 发布者 Agent → 结束
(检索5篇新闻)  (逐篇LLM摘要)  (编排Markdown报告)
```

这是一个**纯线性流水线**，没有条件路由、没有循环，是最简洁的多 Agent 架构。

## 三个 Agent 角色

| Agent | 职责 | 输入 | 输出 |
|---|---|---|---|
| **NewsSearcher（搜索者）** | 检索 AI/ML 最新新闻 | 无 | 文章列表 |
| **Summarizer（摘要者）** | 逐篇生成通俗摘要 | 文章列表 | 摘要列表 |
| **Publisher（发布者）** | 编排成 Markdown 报告 | 摘要列表 | 格式化报告 |

## 关键概念标注

| 概念 | 在代码中的位置 | Java 类比 |
|---|---|---|
| **Pydantic 数据模型** | `Article(BaseModel)` | POJO / DTO |
| **State 状态流转** | `GraphState(TypedDict)` — articles → summaries → report | 责任链模式 |
| **线性流水线** | `search → summarize → publish` | Pipeline / Chain |
| **Agent 类封装** | 每个 Agent 一个类，独立职责 | Service 层 |

## 本课特点

1. **最简洁的多 Agent 架构**：纯线性，没有条件分支和循环，适合入门
2. **三 Agent 分工明确**：搜索、摘要、发布，各司其职
3. **Markdown 报告生成**：LLM 编排最终报告，保存为文件
4. **对比第33课**：第33课有自适应循环和筛选，本课更简洁直接
