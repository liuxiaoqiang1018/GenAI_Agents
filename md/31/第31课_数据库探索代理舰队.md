# 第31课：数据库探索代理舰队（DataScribe）

## 课程概述

本课构建一个**多 Agent 协作的数据库探索系统**：Supervisor 主管调度 4 个子 Agent（分类器、发现者、规划者、推理者），根据用户输入自动分类 → 发现数据库 Schema → 制定查询计划 → 执行 SQL → 生成回答。

**核心模式**：Supervisor 主管模式 + 多 Agent 舰队 + 数据库 Schema 图（NetworkX）

## 架构流程

```
用户输入 → [输入分类]
  ├─ 闲聊/问候/告别 → 直接生成友好回复 → 结束
  └─ 数据库查询 → [Schema已发现？]
                    ├─ 否 → DiscoveryAgent 发现Schema(一次性) → 生成NetworkX图
                    └─ 是 → 跳过
                          ↓
                    PlannerAgent 制定查询计划
                          ↓
                    InferenceAgent 逐步执行SQL
                          ↓
                    SupervisorAgent 生成最终回复 → 结束
```

## 四大 Agent 角色

| Agent | 职责 | Java 类比 |
|---|---|---|
| **SupervisorAgent（主管）** | 调度其他 Agent，生成最终回复 | 编排 Controller |
| **DiscoveryAgent（发现者）** | 探索数据库表结构、字段、外键，生成关系图 | Schema Inspector |
| **PlannerAgent（规划者）** | 将用户问题拆解为可执行的步骤计划 | 任务拆解器 |
| **InferenceAgent（推理者）** | 执行 SQL 查询，返回数据结果 | DAO / Repository |

## 关键概念标注

| 概念 | 在代码中的位置 | Java 类比 |
|---|---|---|
| **Supervisor 模式** | `SupervisorAgent` 持有其他 3 个 Agent 实例 | 服务编排层 |
| **输入分类** | `classify_user_input()` — LLM 判断输入类型 | 路由网关 |
| **Schema 发现** | `DiscoveryAgent.discover()` — 一次性探索，缓存到 State | 元数据扫描 |
| **NetworkX 图** | 表=节点，字段=节点，外键=边 | JGraphT / ER图 |
| **查询规划** | `PlannerAgent.create_plan()` — 拆解为 Inference/General 步骤 | 任务分解 |
| **计划执行** | `SupervisorAgent.execute_plan()` — 遍历步骤调用 InferenceAgent | 任务调度 |
| **State 缓存** | `db_graph` 字段用 reducer 保持不变，避免重复发现 | 缓存 / Singleton |

## 本课特点

1. **4 Agent 协作**：第一次出现完整的 Supervisor + 多专业 Agent 舰队
2. **一次性发现 + 缓存**：Schema 只探索一次，后续查询复用（State 的 reducer 机制）
3. **NetworkX 图**：数据库结构用图表示，查询时用图分析增强 Prompt
4. **计划-执行分离**：先规划再执行，类似第13课但多了"谁来执行"的分配
5. **对话类型分类**：闲聊和数据库查询走不同路径
