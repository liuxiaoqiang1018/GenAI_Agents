# 第16课 - GIF动画生成器（提示词链 + 多模态流水线）

## 环境准备

```bash
pip install langgraph langchain-core httpx python-dotenv
```

`main_debug.py` 只需要 `httpx` 和 `python-dotenv`。

注：原始 notebook 使用 DALL-E 生成图片并合成 GIF。本课因为无 DALL-E API，改为生成详细的**图像提示词**展示流水线模式。在生产环境中可接入任意图像生成 API。

## 概述

本课构建一个**GIF动画生成器**，完整的创作流水线：

1. **角色描述** → 从用户输入生成详细的角色/场景描述
2. **剧情生成** → 基于角色描述，生成5帧动画的剧情
3. **提示词生成** → 为每一帧生成具体的图像提示词
4. **图像生成** → 用提示词调用图像API生成图片（本课模拟）
5. **合成GIF** → 把所有图片拼成GIF动画（本课模拟）

核心模式：**提示词链式传递** —— 每一步的输出作为下一步的输入，前面的"角色描述"会被传递到所有后续步骤以保持**一致性**。

## 动机

创意生成需要**多步骤协作**：
- 写故事的人不一定画得好
- 画画的人需要看故事才能画
- 做动画的人需要所有帧的图片

AI 流水线把这些角色串联起来，每一步把自己的输出传给下一步。

## 架构

```
用户输入："一只戴礼帽的猫在写信"
   │
   ▼
┌──────────────┐
│ 1. 角色描述    │  → 详细的角色外观特征
└──────────────┘
   │ (角色描述传递下去 ↓)
   ▼
┌──────────────┐
│ 2. 剧情生成    │  → 5帧动画的故事线
└──────────────┘
   │ (角色描述 + 剧情 传递下去 ↓)
   ▼
┌──────────────┐
│ 3. 提示词生成  │  → 5个具体的图像提示词
└──────────────┘
   │
   ▼
┌──────────────┐
│ 4. 图像生成    │  → 5张图片（模拟）
└──────────────┘
   │
   ▼
┌──────────────┐
│ 5. 合成GIF    │  → 最终动画
└──────────────┘
```

## 核心概念

### 1. 提示词链式传递（Prompt Chaining）

每一步的 prompt 包含**前面所有步骤的输出**：

```python
# 第1步：只有用户输入
prompt1 = f"根据'{query}'，描述角色外观..."
character_desc = call_llm(prompt1)

# 第2步：用户输入 + 角色描述
prompt2 = f"根据'{query}'，角色：{character_desc}，生成5帧剧情..."
plot = call_llm(prompt2)

# 第3步：角色描述 + 剧情 → 保持一致性
prompt3 = f"角色：{character_desc}，剧情：{plot}，生成图像提示词..."
prompts = call_llm(prompt3)
```

**Java 类比**：
```java
// 像 Builder 模式的链式调用
GifResult result = pipeline
    .describeCharacter(query)           // 每步产出
    .generatePlot(query, charDesc)      // 都传给下一步
    .createPrompts(charDesc, plot)      // 保持上下文一致
    .generateImages(prompts)
    .assembleGif(images)
    .build();
```

### 2. 一致性约束（Consistency Constraint）

GIF的每一帧必须是同一个角色。通过在每个图像提示词中**重复角色特征**来保证一致性：

```
每个提示词 = 角色特征（固定） + 当前帧动作（变化）
```

这是多模态AI的核心挑战：不同帧之间如何保持角色一致。

### 3. 与前几课的对比

| 维度 | 第4课（文本流水线） | 第11课（旅行规划） | **第16课（GIF生成）** |
|------|-------------------|-------------------|---------------------|
| 流水线类型 | 分析型 | 收集→生成 | **创作型链式** |
| 上下文传递 | 单向（前→后） | 累积 | **链式（每步包含前面所有）** |

## 关键洞察

1. **提示词链 = 上下文不断丰富**：第1步的输出喂给第2步，第1+2步的输出喂给第3步，越来越丰富
2. **一致性靠重复**：每个图像提示词都要重复角色特征，因为图像生成模型没有"记忆"
3. **线性但不简单**：虽然图结构是直线，但每步的 prompt 工程（如何传递上下文、如何保持一致性）才是核心价值
4. **模态转换是关键节点**：从文本到图像的跳转（第3→4步）是最关键的，提示词质量直接决定图片质量
