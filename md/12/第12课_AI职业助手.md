# 第12课 - AI职业助手（两级路由 + 多轮对话）

## 环境准备

```bash
pip install langgraph langchain-core httpx python-dotenv
```

`main_debug.py` 只需要 `httpx` 和 `python-dotenv`。

## 概述

本课构建一个**AI职业助手**，帮助用户进行AI领域的职业发展。系统包含6个功能模块，通过**两级路由**分发用户请求：

**第一级路由**（4选1）：
1. **学习资源** → 进入第二级路由
2. **简历制作** → 直接处理
3. **面试准备** → 进入第二级路由
4. **求职搜索** → 直接处理

**第二级路由**：
- 学习 → **教程生成** / **问答对话**（2选1）
- 面试 → **面试题库** / **模拟面试**（2选1）

## 动机

职业助手是一个典型的**多功能系统**，需要：
- 根据用户意图路由到不同模块
- 部分模块需要进一步细分（层级路由）
- 多数模块需要多轮对话（不是一问一答）

这比第9课（客服路由，单级4选1）更复杂，引入了**层级路由**和**多轮对话**。

## 核心概念

### 1. 两级条件路由（Hierarchical Routing）

第一级把请求分成4大类，第二级对"学习"和"面试"做更细的分类：

```python
# 第一级路由
workflow.add_conditional_edges("categorize", route_query, {
    "handle_learning": "handle_learning",       # → 进入第二级
    "handle_resume": "handle_resume",           # → 直接处理
    "handle_interview": "handle_interview",     # → 进入第二级
    "handle_job_search": "handle_job_search",   # → 直接处理
})

# 第二级路由（学习）
workflow.add_conditional_edges("handle_learning", route_learning, {
    "tutorial": "tutorial",
    "qa_bot": "qa_bot",
})

# 第二级路由（面试）
workflow.add_conditional_edges("handle_interview", route_interview, {
    "interview_questions": "interview_questions",
    "mock_interview": "mock_interview",
})
```

**Java 类比**：
```java
// 像 Spring 的两级 DispatcherServlet
@Controller("/api")
public class MainRouter {
    @RequestMapping("/learn/**")   // → 转发到 LearningController
    @RequestMapping("/resume/**")  // → 直接处理
    @RequestMapping("/interview/**") // → 转发到 InterviewController
    @RequestMapping("/job/**")     // → 直接处理
}

@Controller("/api/learn")
public class LearningController {
    @RequestMapping("/tutorial")   // 教程
    @RequestMapping("/qa")         // 问答
}
```

### 2. 多轮对话（Multi-Turn Conversation）

多数处理节点不是"一问一答"，而是维护对话历史的循环：

```python
def qa_bot(state):
    messages = [system_message]
    while True:
        response = call_llm(messages)
        print(f"AI: {response}")
        user_input = input("你: ")
        if user_input == "exit":
            break
        messages.append({"role": "assistant", "content": response})
        messages.append({"role": "user", "content": user_input})
```

**Java 类比**：像 WebSocket 会话 —— 连接建立后持续双向通信，直到客户端断开。

### 3. Few-Shot 分类（Few-Shot Classification）

用几个示例教 LLM 如何分类：

```python
prompt = """分类以下用户请求：
1: 学习AI技术
2: 简历制作

4: 求职搜索

- "帮我写简历" → 2
- "面试会问什么？" → 3
- "有什么AI岗位？" → 4

```

## 关键洞察

1. **两级路由 = 两次 LLM 分类**：第一次分4大类，第二次在子类里再分2小类，一共调2次LLM做分类
2. **多轮对话就是 while 循环**：维护 messages 列表，每轮追加用户和AI的消息，`exit` 退出
3. **不是每个分支都需要二级路由**：简历和求职直接处理，只有学习和面试需要进一步细分
4. **对话历史需要裁剪**：对话太长会超出上下文窗口，需要保留最近的N条消息
