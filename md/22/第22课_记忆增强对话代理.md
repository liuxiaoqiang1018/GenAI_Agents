# 第22课 - 记忆增强对话代理（短期记忆 + 长期记忆）

## 环境准备

```bash
pip install httpx python-dotenv
```

本课不使用 LangGraph，聚焦于**记忆机制**本身。

## 概述

本课构建一个**记忆增强的对话代理**，具备两种记忆：

1. **短期记忆**（对话历史）→ 当前会话的所有消息，让 AI 能记住"刚才说了什么"
2. **长期记忆**（提取的事实）→ 从对话中提取的关键信息，让 AI 能记住"用户是谁、喜欢什么"

核心模式：**双层记忆** —— 短期记忆解决"上下文连贯"，长期记忆解决"跨轮次记忆"。

## 动机

传统聊天机器人的问题：
- **无记忆**：每次对话都从零开始，不知道你是谁
- **纯短期**：只记得当前会话，重启就忘了
- **全量传递**：把所有历史塞进 prompt，很快超出上下文窗口

解决方案：
- **短期记忆**：当前会话的 messages 列表（自动裁剪）
- **长期记忆**：从对话中提取关键事实，精炼存储

## 架构

```
用户输入
   │
   ▼
┌─────────────────────────────────────┐
│              Prompt 组装             │
│  ┌─────────────────────────────┐    │
│  │ System: 你是AI助手           │    │
│  │ System: 长期记忆: {facts}    │ ← 长期记忆注入
│  │ History: [msg1, msg2, ...]  │ ← 短期记忆（对话历史）
│  │ Human: {当前输入}            │    │
│  └─────────────────────────────┘    │
└─────────────────────────────────────┘
   │
   ▼
┌──────────┐
│  LLM调用  │
└──────────┘
   │
   ├──→ 回复用户
   │
   ├──→ 追加到短期记忆（messages列表）
   │
   └──→ 提取关键信息 → 更新长期记忆
```

## 核心概念

### 1. 短期记忆（Short-Term Memory）

就是 messages 列表，每轮对话追加 user + assistant 消息：

```python
messages = [
    {"role": "user", "content": "我叫张三"},
    {"role": "assistant", "content": "你好张三！"},
    {"role": "user", "content": "今天天气怎样？"},
    {"role": "assistant", "content": "..."},
]
```

**Java 类比**：`HttpSession` —— 请求间共享数据，会话结束就没了。

### 2. 长期记忆（Long-Term Memory）

从对话中提取的关键事实，精炼存储：

```python
long_term_memory = [
    "用户名字是张三",
    "用户喜欢晴天",
    "用户是程序员",
]
# 注入到 system prompt 中
system = f"长期记忆: {'. '.join(long_term_memory)}"
```

**Java 类比**：`Redis / Database` —— 持久化存储，跨会话可用。

### 3. 记忆裁剪

短期记忆不能无限增长（会超出上下文窗口），需要裁剪：

```python
if len(messages) > MAX_HISTORY:
    messages = [system_msg] + messages[-MAX_HISTORY:]
```

长期记忆也要限制数量，保留最近N条。

### 4. 会话隔离（Session Management）

不同用户/会话有独立的记忆空间：

```python
chat_store = {}       # session_id → messages列表
long_term_memory = {} # session_id → 事实列表
```

**Java 类比**：`@SessionScope` 的 Bean —— 每个会话独立实例。

### 5. 与前几课的对比

| 维度 | 第1课（简单对话） | 第12课（多轮对话） | **第22课（记忆增强）** |
|------|-----------------|-------------------|---------------------|
| 短期记忆 | 无 | messages列表 | **messages + 自动裁剪** |
| 长期记忆 | 无 | 无 | **事实提取 + 注入prompt** |
| 会话管理 | 无 | 无 | **session_id隔离** |
| 记忆持久化 | 无 | 无 | **可扩展到数据库** |

## 关键洞察

1. **短期记忆 = messages 列表**：就是往数组里 append，超长就裁剪
2. **长期记忆 = 提取的事实列表**：不是存原始对话，而是提炼出关键信息
3. **记忆注入 = 塞进 system prompt**：长期记忆作为上下文注入，LLM 就"记住了"
4. **会话隔离 = 字典按 session_id 存**：不同用户互不影响
5. **这是所有 Agent 的基础设施**：前面学的 Agent 都可以加上记忆系统
