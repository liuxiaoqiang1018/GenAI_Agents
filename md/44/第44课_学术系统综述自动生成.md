# 第44课：学术系统综述自动生成（Systematic Review Automation）

## 课程概述

本课构建一个**全自动学术系统综述生成系统**：输入研究主题，自动规划大纲 → 搜索学术论文(Semantic Scholar API) → 下载PDF → 分析论文内容 → 并行撰写6个章节(摘要/引言/方法/结果/结论/参考文献) → 汇聚成完整论文 → 评审-修改循环 → 输出最终稿。

**核心模式**：规划→搜索→下载→分析→并行写作→汇聚→评审修改循环（最复杂的流水线）

## 架构流程

```
主题输入 → [规划] 拟定综述大纲
            ↓
        [搜索] LLM生成查询 → Semantic Scholar API
            ↓
        [决策] LLM筛选论文URL
            ↓
        [下载] 下载PDF论文
            ↓
        [分析] 逐篇提取关键信息
            ↓
        [并行写作] 6个节点同时撰写:
        ↙  ↙  ↓  ↓  ↘  ↘
    摘要 引言 方法 结果 结论 参考文献
        ↘  ↘  ↓  ↓  ↙  ↙
        [汇聚] 合并为完整论文
            ↓
        [评审] LLM 评审打分
            ↓
        [条件路由]
        ├─ 需修改 → [修改] → 回到评审（最多2轮）
        └─ 通过/达到上限 → [定稿] → 结束
```

## 关键概念标注

| 概念 | 在代码中的位置 | Java 类比 |
|---|---|---|
| **学术搜索工具** | `AcademicPaperSearchTool` — Semantic Scholar API | REST客户端 |
| **工具调用** | `research_node` → LLM决定调用搜索工具 | Function Calling |
| **PDF解析** | `pymupdf4llm.to_markdown()` — PDF转Markdown | Apache PDFBox |
| **并行写作** | 6个write节点同时执行，都从paper_analyzer出发 | CompletableFuture.allOf() |
| **汇聚节点** | `aggregator` — 6路汇聚为一篇完整论文 | Fork/Join |
| **评审循环** | `critique` → `exists_action` → 修改/定稿 | 代码审查循环 |
| **消息过滤** | `get_relevant_messages()` — 过滤工具消息 | 消息过滤器 |

## 本课特点

1. **节点数最多**：20个节点，是所有课中图最大的
2. **6路并行写作**：摘要/引言/方法/结果/结论/参考文献同时生成
3. **真实学术API**：Semantic Scholar API 搜索真实论文
4. **PDF下载解析**：下载PDF论文并用 pymupdf4llm 转为文本分析
5. **评审-修改循环**：写完后评审，不满意就修改（最多2轮）
