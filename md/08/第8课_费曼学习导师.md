# 第8课 - Chiron 费曼学习导师（人机交互 + 多路由）

## 环境准备

```bash
pip install langgraph langchain-core httpx python-dotenv
```

`main_debug.py` 只需要 `httpx` 和 `python-dotenv`。

## 概述

Chiron 是一个基于**费曼学习法**的 AI 导师。费曼学习法的核心：**如果你不能用简单的话解释一个概念，说明你还没真正理解它**。

系统流程：
1. 根据学习主题生成**检查点**（Checkpoint）
2. 针对每个检查点**出题**考查
3. **验证**用户回答（打分 0~1）
4. 理解度 < 70% → 用费曼方法**教学**（简化解释 + 类比）
5. 进入**下一个检查点**，循环直到全部完成

## 动机

与前几课的区别：
- 第6课（ATLAS）：Agent 替你做事（生成计划、笔记）
- **本课（Chiron）**：Agent 和你**交互学习**——出题、等你回答、评估、教你

核心新概念：**Human-in-the-Loop**（人在回路中）——图执行到某个节点时暂停，等用户输入后继续。

## 架构

```
生成检查点
   │
   ▼
处理学习材料（分块/搜索补充）
   │
   ▼
┌─────────────────────────────────────┐
│  检查点循环（每个检查点重复）         │
│                                      │
│  出题 → 等用户回答 → 验证答案        │
│              │                       │
│         理解度 ≥ 70%   理解度 < 70%  │
│              │              │        │
│         下一个检查点    费曼教学      │
│              │              │        │
│              └──────────────┘        │
│                                      │
│  最后一个检查点完成 → 结束            │
└─────────────────────────────────────┘
```

## 核心概念

### 1. Human-in-the-Loop（人在回路中）

```python
# LangGraph 支持在节点间"暂停"等用户输入
graph = workflow.compile(interrupt_before=["user_answer"])
# 执行到 user_answer 节点前暂停
# 用户输入后，更新 state，继续执行
```

**Java 类比**：像审批工作流——流程到某一步暂停，等人审批后继续。
```java
// Activiti/Camunda 工作流
taskService.complete(taskId, Map.of("approved", true));
```

### 2. 检查点循环（Checkpoint Loop）

不是简单的 A→B→C，而是**在一组检查点上循环**：
- `next_checkpoint` 节点把 `current_checkpoint + 1`
- 如果还有检查点 → 回到 `generate_question`
- 如果全部完成 → END

**Java 类比**：`for (int i = 0; i < checkpoints.size(); i++) { ... }`

### 3. 条件评估路由（Verification Routing）

```python
def route_verification(state):
    if state['understanding_level'] < 0.7:
        return 'teach_concept'  # 不及格 → 教学
    if has_next_checkpoint:
        return 'next_checkpoint'  # 及格 → 下一个
    return END                    # 全部完成
```

三路分支：教学 / 下一个 / 结束。

### 4. 费曼教学（Feynman Teaching）

当学生理解度不足时，LLM 用费曼方法生成：
- `simplified_explanation`：用简单语言重新解释
- `key_concepts`：核心概念列表
- `analogies`：生活中的类比

## 与前几课的对比

| 维度 | 第6课（ATLAS） | 第7课（论文助手） | 本课（Chiron） |
|------|---------------|-----------------|---------------|
| 交互模式 | 用户提问一次，Agent处理 | 用户提问一次，Agent处理 | **多轮交互，出题等回答** |
| 循环类型 | Agent并行 | Agent↔Tools | **检查点逐个循环** |
| 人的角色 | 发起者 | 发起者 | **参与者（回答问题）** |
| 评估 | 无 | Judge自我评估 | **对用户的回答评估** |
| 教学 | 无 | 无 | **费曼方法教学** |

## 关键洞察

1. **Human-in-the-Loop 是让 AI 成为"导师"而非"工具"的关键**——Agent 不只是输出结果，还要和用户互动
2. **检查点模式让学习有结构**——不是漫无目的地聊，而是按计划逐步验证
3. **条件路由决定教学策略**——及格就前进，不及格就教学，很像真实的老师
4. **费曼方法是 Prompt Engineering 的好场景**——让 LLM 用简单语言和类比解释复杂概念
